# Лабораторная работа 5: Аутентификация и Авторизация (Собственная + Keycloak)

## Запуск инфраструктуры
Для локального запуска Keycloak в режиме разработки выполните команду:
```bash
docker-compose up -d
```
Keycloak будет доступен по адресу: `http://localhost:8081` 
Учетные данные администратора по умолчанию: `admin` / `admin`

## Сравнительный анализ: Собственная авторизация vs Keycloak

| Критерий | Собственная система (Custom Auth) | Keycloak (Identity Provider) |
|---|---|---|
| **Время разработки** | Высокое: требуется самостоятельная реализация токенов, хранения паролей (шифрование, соли), форм логина/регистрации, восстановления пароля и т.д. | Низкое (интеграция): формы и логика делегированы стороннему сервису. Достаточно подключить OAuth2/OIDC клиента. |
| **Безопасность** | Средняя: Возможны уязвимости из-за ошибок реализации (XSS, некорректная валидация JWT, утечка ключей). | Высокая: продукт поддерживается сообществом RedHat, регулярно проходит аудит, поддерживает MFA "из коробки" (2FA, WebAuthn). |
| **Гибкость и кастомизация** | Максимальная: можно встроить логику любой сложности в бизнес-процесс. | Высокая, но ограничена архитектурой (SPIs): Кастомизация тем и логин-потоков требует изучения документации и написания модулей на Java. |
| **Единый вход (SSO)** | Нет "из коробки" (или очень сложно реализуется надежно). | Да, нативная поддержка. Легко добавить вход через Google, GitHub и т.д. |
| **Управление пользователями** | Требуется разрабатывать административную панель с нуля. | Готовая и мощная админка (настройка ролей, групп, атрибутов, сессий). |
| **Обслуживание и поддержка** | Упрощенный деплой (одно приложение), но постоянная поддержка кода безопасности. | Усложненный деплой (Application + Keycloak + Database), но поддержка безопасности перекладывается на Keycloak. |

### Вывод
Для простых изолированных проектов может подойти самописная система (например, на основе JWT). Однако для корпоративных решений, микросервисов и масштабируемых систем, требующих стандартизации (OAuth2.0 / OpenID Connect) и поддержки SSO, использование Keycloak или аналогичных решений (Auth0, Okta) является предпочтительным.
