# Отчет по лабораторной работе №1

## Игра "Память" на Android

### Выполненные требования

#### ✅ Минимальные требования (на оценку 4)

1. **Игровое поле 4х4**
   - Реализовано с помощью `GridView` с 4 колонками
   - Всего 16 карточек (8 пар)
2. **Отображение не более одной картинки**
   - В каждый момент времени открыта максимум одна карточка
   - Логика проверяется в методе `handleCardClick()` в MainActivity
   - Используется переменная `previousCard` в классе `MemoryGame`

3. **Парное удаление при совпадении**
   - Реализовано в методе `checkCard()` класса `MemoryGame`
   - При совпадении карточки помечаются как `isMatched = true`
   - Найденные пары затемняются (прозрачность 0.3f)

4. **Перезапуск игры**
   - Кнопка "Новая игра" на главном экране
   - Диалог подтверждения для предотвращения случайного перезапуска

#### ⭐ Бонусные функции (на оценку 5+)

1. **Настройки игрового поля**
   - Переменный размер: 4x4 (8 пар) или 6x6 (18 пар)
   - Настройки сохраняются через `SharedPreferences`
   - Доступ через меню → Настройки
   - Класс `GameSettings` управляет сохранением настроек

2. **Запись информации (статистика)**
   - Количество сыгранных игр
   - Лучшее время прохождения
   - Минимальное количество ходов
   - Все данные сохраняются локально
   - Обновляются автоматически при победе

3. **Оформление игры и информация для пользователя**
   - **Таймер**: отображение времени игры в формате MM:SS
   - **Счетчик ходов**: показывает количество попыток
   - **Счетчик пар**: текущие найденные пары / всего пар
   - **Мотивационные элементы**:
     - Поздравление при победе с эмоджи 🎉
     - Отображение личной статистики
   - **Цветовая схема**: приятный голубой дизайн
   - **Анимация**: задержка 0.6 сек перед закрытием несовпадающих карт

4. **Собственное оформление**
   - 8 уникальных векторных изображений для карточек
   - Разнообразные цвета и формы (круги, квадраты, ромбы, кольца)
   - Красивая рубашка карты (синяя с закругленными углами)
   - Профессиональный Material Design

5. **Смешанный режим (пары + тройки)**
   - Возможность присутствия на игровом поле не только пар, но и троек картинок
   - Переключатель в настройках для активации смешанного режима
   - Когда переключатель активен - на поле присутствуют и пары (2 карточки), и тройки (3 карточки)
   - Когда переключатель неактивен - только выбранный режим (пары или тройки)
   - Автоматический расчёт оптимального распределения пар и троек для размера поля
   - Логика проверки совпадения учитывает размер группы каждой карточки

### Технические детали реализации

#### Архитектура приложения

**Класс Card (Модель карточки)**

```
- int id                    # ID изображения
- int imageResourceId       # Ресурс изображения
- boolean isMatched         # Найдена ли пара
- boolean isRevealed        # Открыта ли карточка
- int groupSize             # Размер группы (2 для пар, 3 для троек)
```

**Класс MemoryGame (Игровая логика)**

```
- List<Card> cards          # Список всех карточек
- Card previousCard         # Предыдущая открытая карточка
- int pairsFound           # Найденные пары/группы
- int moves                # Количество ходов
- boolean mixedMode        # Смешанный режим (пары + тройки)
```

**Класс CardAdapter (Отображение)**

```
- Управляет отображением карточек в GridView
- Показывает рубашку или изображение в зависимости от состояния
- Затемняет найденные пары
```

**Класс GameSettings (Настройки)**

```
- Сохранение размера поля
- Хранение статистики
- Режим игры (пары/тройки/смешанный)
- Использование SharedPreferences
```

**MainActivity (Главная активность)**

```
- Управление UI
- Обработка кликов
- Таймер игры
- Диалоги
- Меню настроек
```

#### Основной игровой цикл

1. Пользователь нажимает на карточку
2. Проверка: открыта/найдена ли карточка?
3. Если это первая карточка - открыть и запомнить
4. Если это вторая карточка:
   - Открыть её
   - Проверить совпадение с первой
   - Если совпадают - пометить обе как найденные
   - Если не совпадают - закрыть обе через 0.6 сек
5. Обновить счетчики
6. Проверить, все ли пары найдены

#### Особенности реализации

1. **Handler и задержки**
   - Используется для таймера (обновление каждую секунду)
   - Для задержки перед закрытием несовпадающих карт
   - Правильная очистка в `onDestroy()`

2. **SharedPreferences**
   - Хранение настроек между запусками
   - Накопление статистики
   - Отслеживание рекордов

3. **AlertDialog**
   - Красивые диалоги для взаимодействия
   - Подтверждение перезапуска
   - Отображение результатов
   - Настройки и статистика

4. **Векторная графика**
   - XML drawable для всех изображений
   - Масштабируется без потери качества
   - Легкий вес приложения
